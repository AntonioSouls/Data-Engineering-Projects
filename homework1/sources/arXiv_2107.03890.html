Perspective-n-Point-and-Line (Pnğ‘›nPL) algorithms aim at fast, accurate, and robust camera localization with respect to a 3D model from 2D-3D feature correspondences, being a major part of modern robotic and AR/VR systems. Current point-based pose estimation methods use only 2D feature detection uncertainties, and the line-based methods do not take uncertainties into account. In our setup, both 3D coordinates and 2D projections of the features are considered uncertain. We propose Pnğ‘›nP(L) solvers based on EPnğ‘›nPÂ [20] and DLSÂ [14] for the uncertainty-aware pose estimation. We also modify motion-only bundle adjustment to take 3D uncertainties into account. We perform exhaustive synthetic and real experiments on two different visual odometry datasets. The new Pnğ‘›nP(L) methods outperform the state-of-the-art on real data in isolation, showing an increase in mean translation accuracy by 18% on a representative subset of KITTI, while the new uncertain refinement improves pose accuracy for most of the solvers, e.g. decreasing mean translation error for the EPnğ‘›nP by 16% compared to the standard refinement on the same dataset. 1 Introduction

Camera localization using sparse feature correspondences is a major part of augmented or virtual reality and robotic systems. The Perspective-nğ‘›n-Point(-and-Line), or Pnğ‘›nP(L), methods can be successfully used to estimate the pose of a calibrated camera from sparse feature correspondences. Line features can increase localization accuracy in man-made self-similar environments which lack surfaces with distinctive texturesÂ [36, 43, 15, 29], motivating the use of Pnğ‘›nP(L) methodsÂ [41].

While vision-based localization with respect to an automatically reconstructed map of sparse features is an important part of current robotic and AR/VR systems, commonly used Pnğ‘›nP methods treat the features as absolutely accurateÂ [47, 14, 20, 16]. The more recent 2D covariance-aware methods relax this assumptionÂ [40, 6], but only for the feature detections, still assuming perfect accuracy of the 3D feature coordinates.

In maps reconstructed with structure-from-motion, 3D feature coordinate accuracy can vary. Stereo triangulation errors grow quadratically with respect to the object-to-sensor distance, so the accuracy in estimating point depth can vary in several orders of magnitude, while the line stereo triangulation accuracy depends on the angle between the line direction and the baseline. Nevertheless, to the best of our knowledge, no prior method for Pnğ‘›nP(L) was designed to take both 3D and 2D uncertainties into account.

We propose to integrate the feature uncertainty into the Pnğ‘›nP(L) methods, see Fig.Â 1, which is our main contribution. We build on the classical DLSÂ [14] and EPnğ‘›nPÂ [20] methods. Additionally, we propose a modification to the standard nonlinear refinement, which is normally used after the Pnğ‘›nP(L) solver, to take 3D uncertainties into account. An exhaustive evaluation on synthetic data and on the two real indoor and outdoor datasets demonstrates that new Pnğ‘›nP(L) methods are significantly more accurate than state-of-the-art, both in isolation and in a complete pipeline, e.g. the proposed DLSU method reduces the mean translation error on KITTI by 18%, see SectionÂ 4. The proposed uncertain pose refinement can improve the pose accuracy by up to 16% in exchange of an extra 5-10% of the computational time. In a synthetic setting with noise in 2D feature detections, the new methods have the same accuracy as the most accurate 2D uncertainty-aware methodsÂ [6, 40]. The code is available at https://alexandervakhitov.github.io/uncertain-pnp/.

2 Related work

We start with discussing how proposed methods relate to known Pnğ‘›nP methods for arbitrary number of correspondences nğ‘›n, Pnğ‘›nPL and 2D covariance-aware methods.

Perspective-nğ‘›n-point.

Geometric gold standard costÂ [13] for pose estimation for arbitrary nğ‘›n is highly non-convex, and direct pose solvers rely on simplified algebraic costs. Early methodsÂ [28, 22, 35, 31, 8, 4, 1] were slow and inaccurate.

Starting fromÂ [24] fast direct solvers were developedÂ [21, 14, 16, 47]. EPnğ‘›nPÂ [24] was the first to provide fast and accurate pose estimate solving a least-squares system with nonlinear constraints.

EPnğ‘›nP was developed further inÂ [20, 7, 6, 41]: [20] proposed adaptive PCA-based choice of control points and a fast iterative refinement step, [7] designed an EPPnğ‘›nP method for robust pose estimation in presence outliers. Structure-from-motionÂ [34], visual SLAMÂ [25], object pose estimationÂ [38] rely on EPnğ‘›nP due to its robustness and fast computational time. A proposed solver EPnğ‘›nPU is a derivative of EPnğ‘›nP, providing better accuracy when feature uncertainty information is available.

Groebner basis solvers for polynomial systems are more accurate but are computationally more demandingÂ [14, 48, 47, 16, 12]. The DLS methodÂ [14] is fast enough for real-time use and has higher accuracy compared to the EPnğ‘›nP, while the most accurate method OPnğ‘›nPÂ [47] is significantly slower. DLS minimizes the object space error and uses the Cayley parameterization, and has a singularity which can be avoidedÂ [27]. In this work, we build on the DLS and take feature uncertainties into account, however relying on the OPnğ‘›nP-like algebraic cost instead of the object space error.

Pnğ‘›nPL methods.

An early DLT methodÂ [13] as well as an algorithmÂ [4] can compute camera pose from nğ‘›n line correspondences, but have inferior accuracy compared to new polynomial solver-based approachesÂ [23, 30, 45, 18, 49]. Extending EPnğ‘›nP or OPnğ‘›nP to Pnğ‘›nPLÂ [41, 42] is practical since a Pnğ‘›nPL method uses all available mixed correspondences at once. In this work, we propose EPnğ‘›nPLU/DLSLU methods, which take line uncertainty into account in order to improve the pose estimates.

Pnğ‘›nP with uncertainty.

Features are detected with varying uncertainty, and 2D uncertainty-aware Pnğ‘›nP methodsÂ [6, 40] use it to improve the estimated pose accuracy. CEPPnğ‘›nPÂ [6] builds on EPPnğ‘›nP and inherits the base methodâ€™s low computational complexity. MLPnğ‘›nPÂ [40]

is more accurate and computationally demanding than CEPPnğ‘›nP, because it combines both a linear solver and a refinement into one method.

Both CEPPnğ‘›nP and MLPnğ‘›nP use only 2D feature detection uncertainty and work only for points.

In contrast, the approach we present in this paper is more accurate due to the use of both 2D and 3D feature uncertainty and works for a mixed set of line and point correspondences.
