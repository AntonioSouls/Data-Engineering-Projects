Abstract

Efficient point cloud representation is a fundamental element of Lidar-based 3D object detection. Recent grid-based detectors usually divide point clouds into voxels or pillars and construct single-stream networks in Birdâ€™s Eye View. However, these point cloud encoding paradigms underestimate the point representation in the vertical direction, which cause the loss of semantic or fine-grained information, especially for vertical sensitive objects like pedestrian and cyclists. In this paper, we propose an explicit vertical multi-scale representation learning framework, VPFusion, to combine the complementary information from both voxel and pillar streams. Specifically, VPFusion first builds upon a sparse voxel-pillar-based backbone. The backbone divides point clouds into voxels and pillars, then encodes features with 3D and 2D sparse convolution simultaneously. Next, we introduce the Sparse Fusion LayerÂ (SFL), which establishes a bidirectional pathway for sparse voxel and pillar features to enable the interaction between them. Additionally, we present the Dense Fusion Neck (DFN) to effectively combine the dense feature maps from voxel and pillar branches with multi-scale. Extensive experiments on the large-scale Waymo Open Dataset and nuScenes Dataset demonstrate that VPFusion surpasses the single-stream baselines by a large margin and achieves state-of-the-art performance with real-time inference speed.

1 Introduction

LiDAR-based 3D object detection methods have been widely adopted in autonomous driving and robot navigation systems, as point clouds from LiDAR sensors explicitly reflect geometric information and are rarely affected by weather conditions. However, unlike 2D images, point clouds are sparse and non-uniformly distributed, which brings two challenges for 3D object detection: Constructing efficient detection network and establishing robust object representation from points with divergent distributions.

There are several paradigms for point cloud representation. Among which, grid-based methodsÂ [60, 23, 21, 45, 22] quantize the point cloud into regular voxels or pillars and generate bounding boxes in 2D Birdâ€™s Eye View (BEV), enables the usage of mature convolution architectureÂ [30, 31]. Though superior to Range-view detectorsÂ [36, 2, 15] or point-based methodsÂ [42, 47, 57, 62] in both performance and efficiency, we argue that existing grid-based methods are insufficient at vertical point cloud representation learning, causing the loss of semantic or fine-grained information. Specifically, for detectors deploy voxels, commonly used 3D sparse submanifold convolutionÂ [19] restricts the feature diffusion from non-empty voxel to empty voxel, which leads to the limited receptive field in the vertical direction. On the contrary, pillar-based methods discretized point clouds into vertical volumes, where a pillar encodes all neighboring points in the specific X-Y coordinate. This paradigm obtains the full-ranged vertical receptive field. However, it is weak in capturing fine-grained features.

To analyze the impact of vertical representation on the grid-based methods, we conduct an exploratory experiment on Waymo Open DatasetÂ [50]. Next, we count the recall VS. vertical sparsity curve of two representative grid-based detectorsÂ [60, 21] at the specific horizontal occupancy or point density. As illustrated in Fig.Â 1, Voxel-based CenterPoint_IoUÂ [60, 23]) is better when the vertical sparsity is high, while pillar-based PillarNetÂ [21] is superior in the sparse vertical distribution. This phenomenon proves the insufficient vertical representation in the voxel or pillar-based methods, while the complementary performance under the variant vertical sparsities encourages us to combine the voxels and pillars for vertical encoding enhancement.

In this paper, we propose Voxel-Pillar Fusion (VPFusion), a single-stage two-stream 3D detector that can capture complementary vertical information from both voxels and pillars. We first design a sparse voxel-pillar-based backbone, which divides the point clouds into voxels and pillars, then encodes sparse volume features with 3D and 2D sparse convolutions simultaneously. To enlarge the vertical receptive field of voxels and enrich the fine-grained information in pillars, we present the Sparse Fusion Layer (SFL) to establish the voxel-pillar bidirectional connection. To be specific, SFL aggregates voxel features in the vertical direction and broadcasts pillar features to corresponding vertical voxels, then fuses the aggregated and broadcasted features with the original pillar and voxel features. Moreover, we present the Dense Fusion Neck (DFN) to combine the multi-scale dense feature maps from voxel and pillar branches. We conduct extensive experiments on nuScenes and Waymo Open Dataset. The results indicate that VPFusion achieves state-of-the-art performance with real-time inference speed, and indeed supplements the vertical representation from both voxels and pillars as demonstrated in Fig.Â 1.

2 Related Work

2.1 Grid-based 3D Object Detection

Detectors Encoded via Voxels. Voxel-based methods divide the point clouds into 3D voxels and deploy 3D convolutions for voxel-wise feature extraction. Pioneering work VoxelNetÂ [67] replaces the hand-crafted representation with the proposed voxel feature encoding layer, enabling the end-to-end training procedure. Then, SECONDÂ [55] introduce the 3D sparse convolutionÂ [18, 19] to avoid the redundant computation on empty voxels. Two-stage detectorsÂ [33, 10, 60] follow the coarse-to-fine pattern in 2D detectionÂ [43, 30], which generate Region of Interests (RoIs) in the first stage and refine the predictions with specific RoI features. For instance, Voxel R-CNNÂ [10] proposes the voxel RoI pooling to extract RoI features from the voxels. As a contrast, single-stage methodsÂ [63, 64, 23] directly generate bounding boxes in one stage. CIA-SSDÂ [63] presents the IoU-aware confidence rectification to make the confidence consistent with regression accuracy. AFDetV2Â [23] proposes a multi-task framework to incorporate keypoint auxiliary supervision with IoU-aware prediction, which achieves competitive results. More recently, some worksÂ [34, 44, 35, 22] explore the powerful transformer architectureÂ [53, 12, 24, 32] in 3D object detection. For example, VoxSeTÂ [22] presents a voxel-based set attention module to model the long-range dependencies in voxelized point clusters. To the best of our knowledge, the limitation of vertical representation learning in voxel-based methods has not been studied.

Pillar-based 3D Object Detection. Pillar-based detectors encode the point clouds in Birdâ€™s Eye View, which directly produce the 2D features, thus improving the computation efficiency. PIXORÂ [56] represents the scene from BEV space, facilitating the application of 2D convolution for object prediction. Then, PointPillarsÂ [26] convert the point clouds to 2D pillars (vertical volumes) and deploys PointNetÂ [40, 41] as the encoder to generate pillar features. The following work HVNetÂ [59] presents the hybrid voxel feature encoding strategy to aggregate multi-scale 2D voxels into unified point-wise features. PillarNetÂ [21] proposes the 2D sparse convolution-based backbone for efficient pillar encoding. It achieves promising performance while running at real-time speed. Finally, SSTÂ [13] analyzes the characteristic of 3D objects in BEV, then proposes a single-stride transformer to boost the performance on the challenging pedestrian category. As mentioned in Sec.Â 1, pillar-based representation brings ambiguity in the vertical direction.

2.2 Multi-Source Feature Fusion

Methods of multi-source fusion are proposed to combine complementary information from different sources. For instance, point-voxel join representationÂ [58, 48, 45, 38] is presented to integrate the flexible receptive field with efficient feature learning schemes. STDÂ [58] generates proposals from point-wise features and deploys the PointPool layer to provide dense voxel features for box refinement. In contrast, PV-RCNNÂ [45] regresses the proposals from a 3D sparse convolution-based encoder, then aggregates voxel features into keypoints via proposed voxel set abstraction for box refinement. Additionally, multi-view fusionÂ [6, 25, 66, 54] is another paradigm for information supplementation. MVFÂ [66] utilizes the complementary information from both BEV and perspective view, while Pillar-ODÂ [54] applies pillar encoder in BEV and cylindrical view. Recently, multi-modal fusionÂ [29, 39, 1, 27] has achieved remarkable progress. CLOCSÂ [39] proposes a late fusion strategy to exploit the geometric and semantic consistencies between 2D and 3D predictions. DeepFusionÂ [27] presents InverseAug and LearnableAlign to align multi-modal features in the late stage. While multi-source fusion provides extra information from different sources, it also raises a problem. As the separate sources usually vary, effective alignment of multi-source features becomes challenging and sophisticated. In our work, we design a simple yet efficient structure for voxel-pillar fusion.

3 Methodology

3.1 Overview

Motivated by enhancing the point cloud vertical representation, we propose Voxel-Pillar Fusion (VPFusion), a single-stage two-stream 3D object detector to capture complementary information from 2D and 3D volumes. As shown in Fig.Â 2, VPFusion first quantizes the point clouds into voxels and pillars with the same resolution in the X-Y plane. Then, we adopt the Sparse Voxel-Pillar-based Backbone (Sec.Â 3.2). It has four stages, each stage is composed of a sparse conv block, where 3D and 2D sparse convolutions form two branches for sparse voxel and pillar feature extraction. To enrich the local context for the pillar branch and introduce the vertical semantic information to the voxel branch, VPFusion deploys the Sparse Fusion Layer (SFL) at the end of each stage, which establishes a bidirectional pathway between the sparse voxel and pillar features (Sec.Â 3.3). Next, Dense Fusion Neck (Sec.Â 3.4) fuses multi-scale sparse features from different branches and generates the dense feature maps in Birdâ€™s Eye View (BEV). Finally, we apply the common center-based detection headÂ [60] with IoU-Aware rectification functionÂ [23] for object prediction (Sec.Â 3.5).

3.4 Dense Fusion Neck

After the sparse backbone, sparse features are converted to dense BEV feature maps. 5 Conclusion

In this paper, we propose VPFusion, a single-stage two-stream 3D detector for robust vertical point cloud representation. VPFusion adpots the sprase voxel-pillar-based backbone for consistent voxel-pillar encoding, and applies the Sparse Fusion Layer and Dense Fusion Neck to efficiently capture complementary information from voxel and pillar branches, resulting in state-of-the-art performance on the Waymo Open Dataset with real-time inference speed.

Limitations. VPFusion relies on consistent occupancy in the birdâ€™s eye view for element-wise fusion, which restricts the voxel and pillar partition to be equal horizontally. For future work, we will explore the asynchronous voxel-pillar fusion strategies (e.g., variant resolution or multi-stride fusion) for more flexible point cloud representation learning.
