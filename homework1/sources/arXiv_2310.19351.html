Semi- and Weakly-Supervised Domain Generalization for Object Detection

Ryosuke Furuta and Yoichi Sato
The University of Tokyo
{furuta, ysato}@iis.u-tokyo.ac.jp
Abstract

Object detectors do not work well when domains largely differ between training and testing data. To solve this problem, domain generalization approaches, which require training data with ground-truth labels from multiple domains, have been proposed. However, it is time-consuming and labor-intensive to collect those data for object detection because not only class labels but also bounding boxes must be annotated. To overcome the problem of domain gap in object detection without requiring expensive annotations, we propose to consider two new problem settings: semi-supervised domain generalizable object detection (SS-DGOD) and weakly-supervised DGOD (WS-DGOD). In contrast to the conventional domain generalization for object detection that requires labeled data from multiple domains, SS-DGOD and WS-DGOD require labeled data only from one domain and unlabeled or weakly-labeled data from multiple domains for training. We show that object detectors can be effectively trained on the proposed settings with the same student-teacher learning framework, where a student network is trained with pseudo labels output from a teacher on the unlabeled or weakly-labeled data. The experimental results demonstrate that the object detectors trained on the proposed settings significantly outperform baseline detectors trained on one labeled domain data and perform comparably to or better than those trained on unsupervised domain adaptation (UDA) settings, while ours do not use target domain data for training in contrast to UDA.

1 Introduction

Object detection has been attracting much attention because it has practically useful applications such as autonomous driving. Recent object detectors have performed tremendously well on commonly used benchmark datasets for object detection, such as MSCOCO [15] and PASCAL VOC [5]. However, the performance of the detectors significantly drops when they are deployed on unseen domains, i.e., when the training and testing domains are different. For example, [10] reported a performance drop caused by the difference in image styles, and [13] showed one caused by the weather and time difference in the images captured by car-mounted cameras.

To solve the problem, many researchers have recently been exploring unsupervised domain adaptive object detection (UDA-OD) [4, 13, 2]. In UDA-OD, we train object detectors using source domain data with ground-truth labels (bounding boxes and class labels) and unlabeled target domain data to adapt the detectors to the target domain. However, in the real world, target domain data cannot always be accessed in the training phase.

Domain generalizable object detection (DGOD) is another popular approach for solving the problem of the performance drop caused by the domain gaps [14, 35]. In DGOD, we train object detectors using labeled data from multiple domains so that the detectors work well on unseen domains. However, it is labor-intensive to collect these data for object detection because both bounding boxes and class labels for all objects in the images must be annotated. Although single-DGOD [32, 6, 25, 31, 28], where we train object detectors to generalize unseen domains using labeled data from one single domain, has recently been tackled, the performance gain is still limited.

In this paper, we propose two new tasks as more realistic settings: i) semi-supervised DGOD (SS-DGOD) and ii) weakly-supervised DGOD (WS-DGOD). As shown in Fig. 1, the goal of SS-DGOD is to generalize object detectors to unseen domains using labeled data only from one single domain and unlabeled data from multiple domains. Note that the target domain data is not included in the training data. In WS-DGOD, we use weakly-labeled data from multiple domains instead of the unlabeled data in SS-DGOD. Here, “weakly-labeled” means that we have only image-level labels that show the existence of each class in each training image and do not have bounding box annotations.

We show that object detectors can be effectively trained on the proposed settings with the same student-teacher learning framework, where a student network is trained with pseudo labels output from a teacher on the unlabeled or weakly-labeled data. However, if we train the detectors naively by the student-teacher learning framework, the detectors are heavily biased by the labeled domain data at the beginning of the training. To suppress the bias, we propose incorporating instance normalization (IN) layers [24], which normalize the image style features in the feature maps, into the detectors.

The experimental results demonstrate that the object detectors trained on the proposed settings perform significantly better than baseline models trained on Single-DGOD and comparably to or better than ones trained on unsupervised domain adaptation settings, while ours do not access target domain data in the training phase. Also, we experimentally show that IN layers boost the performance of the detectors on the proposed settings by suppressing the bias of image styles from the labeled domain. Although Pan et al. [19] and Jing et al. [11] already proposed incorporating IN layers for domain generalization, we found that it does not always improve the performance on Signle-DGOD and DGOD in our experiments. In contrast, our experiments show that IN layers constantly improve the performance on the proposed SS-DGOD and WS-DGOD settings.

Our contributions are summarized as follows:

  • •

    We propose two new problem settings: SS-DGOD and WS-DGOD.

  • •

    We show that object detectors can be effectively trained on the proposed settings with the same student-teacher learning framework.

  • •

    We propose incorporating IN layers to improve the performance of the detectors trained on the proposed settings by suppressing the bias to the labeled domain in the student-teacher learning framework.

  • •

    Our experiments show that the detectors trained on the proposed settings perform comparably to or better than those trained on the related settings.

3 Related Works

3.1 Domain Generalization for Object Detection

Many methods have been proposed for domain generalization for image classification as summarized in recent survey papers [37, 27]. However, domain generalization for object detection has not been explored widely, compared with image classification.

Lin et al. [14] proposed an approach to disentangle domain-specific and domain-invariant features by adversarial learning on both image-level and instance-level features for DGOD. Liu et al. [16] tackled DGOD in underwater object detection and proposed DG-YOLO. For Single-DGOD, Wang et al. [31] proposed a self-training method that utilizes the temporal consistency of objects in videos. Wu et al. [32] proposed a method that disentangles domain-invariant features by contrastive learning and self-distillation. Fan et al. [6] proposed perturbing the channel statistics of feature maps, which can be interpreted as data augmentation of image styles to a variety of domains. Wang et al. [28] proposed a disentangle method on frequency space for object detection from unmanned aerial vehicles. Recently, Vidit et al. [25] proposed an augmentation approach using a pre-trained vision-language model (CLIP) with textual prompts.

Unlike all the above methods, as discussed in Sects. 1 and 2, we propose and tackle new problem settings called SS-DGOD (Semi-Supervised Domain Generalization for Object Detection) and WS-DGOD (Weakly-Supervised Domain Generalization for Object Detection), which are more practical than DGOD. Furthermore, our experiments show that the object detectors trained on SS-DGOD and WD-DGOD outperform the state-of-the-art method for Single-DGOD [25] by utilizing the unlabeled (or weakly-labeled) data.

3.2 UDA for Object Detection

As summarized in [18], various methods have been proposed for UDA-OD. There are two main streams for UDA-OD, i.e., alignment-based [3, 21] and student-teacher learning-based [4, 13], but the recent trend is the latter approach. In this paper, we show that the proposed two new settings (SS-DGOD and WS-DGOD) can be solved by the student-teacher learning framework. Also, we show that the object detectors trained on the proposed settings can perform comparably to or better than those trained on the UDA-OD setting.

Another approach to make detectors robust to unseen domains is an online adaptation using test images [26]. However, this approach requires extra computation for online updates at test time in contrast to our settings.

3.3 Semi-Supervised Domain Generalization

There are a few existing methods that utilize both labeled and unlabeled data for domain generalization (SSDG) on image classification [36, 38]. Zhang et al. [36] proposed an unsupervised pretraining method called DARLING, which performs contrastive learning on unlabeled images to obtain domain-irrelevant feature representation. Zhou et al. [38] extended a semi-supervised learning method called FixMatch [23] to SSDG.

To the best of our knowledge, this paper is the first attempt to tackle SSDG for object detection. In addition, we tackle the “weakly-labeled” setting (i.e., WS-DGOD), which is not defined on image classification.

3.4 Student-Teacher Learning

Several existing works have studied the use of the student-teacher learning frameworks for a variety of tasks such as domain generalization on image classification [33], (in-domain) weakly-supervised object detection [30], (in-domain) semi-supervised object detection [17], (in-domain) semi-supervised image classification [23], and UDA for semantic segmentaion [1, 29, 9, 34]. Note that our aim is not to propose a technically novel method. Instead, we show that the proposed new settings can be effectively solved by adopting the student-teacher learning framework.

4 Training Method

4.1 Overview and Key Idea

In both SS-DGOD and WS-DGOD, our goal is to obtain object detectors that work well on the unseen target domain data 𝒟t={Xt}subscript𝒟𝑡subscript𝑋𝑡\mathcal{D}_{t}=\{X_{t}\}. Gulrajani and Lopez-Paz [7] reported that if carefully implemented, empirical risk minimization (i.e., the image classifier simply trained with supervised learning on multiple domains) outperformed the state-of-the-art domain generalization methods on several benchmark datasets for image classification. Following this knowledge, we aim to train an object detector on multiple domains 𝒟si​(i=1,⋯,ND)subscript𝒟subscript𝑠𝑖𝑖1⋯subscript𝑁𝐷\mathcal{D}_{s_{i}}(i=1,\cdots,N_{D}). However, we have no ground-truth labels (or have only weak labels) for 𝒟si​(i=2,⋯,ND)subscript𝒟subscript𝑠𝑖𝑖2⋯subscript𝑁𝐷\mathcal{D}_{s_{i}}(i=2,\cdots,N_{D}) although ground-truth labels are available for 𝒟s1subscript𝒟subscript𝑠1\mathcal{D}_{s_{1}}. Therefore, the question here is how to train the detector on those domains. Our solution is to adopt the student-teacher learning framework [13, 2] as shown in Fig. 2, where we have two networks (teacher and student) with the same structure and train the student network using the pseudo-labels generated by the teacher network. Note that our training framework can be applied to any object detector although we hereafter describe the loss functions of FasterRCNN [20] as an example for ease of explanation.

4.3.4 Normalize Image Style Features

By the student-teacher learning framework described in Sects. 4.1-4.3, we can effectively train object detectors on both SS-DGOD and WS-DGOD. However, the drawback of student-teacher learning on the two settings is that the networks will be heavily biased to the domain s1subscript𝑠1s_{1} at the beginning of the training because they are initialized by the supervised learning with the labeled data 𝒟s1subscript𝒟subscript𝑠1\mathcal{D}_{s_{1}}. To solve this problem, we propose incorporating IN layers into the networks. As discussed in [12], means and standard deviations of feature maps in CNNs represent styles of input images. Therefore, we suppress the bias to the image styles of the domain s1subscript𝑠1s_{1} by normalizing the means and the standard deviations of the feature maps through IN layers. In the experiments, we show that the IN layers significantly boost the performance of the detectors trained on SS-DGOD and WS-DGOD.

5 Experiments

5.1 Dataset

We used two datasets for the experiments.

  • •

    Car-mounted camera dataset [32]: In this dataset, the domains are defined as different times and weather: daytime-sunny, night-sunny, daytime-foggy, dusk-rainy, and night-rainy. The number of images for each domain is 27,708, 18,310, 2,642, 3,501, and 2,494, respectively. We used daytime-sunny as the labeled domain s1subscript𝑠1s_{1} and used night-sunny and daytime-foggy as the unlabeled (or weakly-labeled) domains s2,s3subscript𝑠2subscript𝑠3s_{2},s_{3}. We used each of the remaining domains (dusk-rainy and night-rainy) as the target domain. Because the train/val/test split is not publicly available for daytime-sunny, dusk-rainy, and night-rainy, we used all images of daytime-sunny, the trainval set of night-sunny, and the trainval set of daytime-foggy for training. We then used the test set of night-sunny and the test set of daytime-foggy for validation. We used all images of dusk-rainy and night-rainy for evaluation (testing). There are seven object classes: bus, bike, car, motor, person, rider, and truck.

  • •

    Artistic style image dataset [10]: This dataset has four domains: natural image, clipart, comic, and watercolor. The natural image domain has 16,551 images from PASCAL VOC07&12, and the other domains have 1,000, 2,000, and 2,000 images, respectively. We set the natural image domain as the labeled domain s1subscript𝑠1s_{1} and set clipart and comic as the unlabeled domains s2,s3subscript𝑠2subscript𝑠3s_{2},s_{3}. We set watercolor as the target domain. Concretely, we used the trainval set of PASCAL VOC 2007&2012, the train set of clipart, and the train set of comic for training. We then used the test sets of clipart and comic for validation. For evaluation (testing), we used the test set of watercolor. There are six object classes (bike, bird, car, cat, dog, and person), and we removed the images that do not contain these classes.

5.2 Implementation Details

We used soft pseudo labeling proposed in [2] for student-teacher learning. As the object detector, we used Gaussian FasterRCNN [2], in which the regression output is modified to adopt the soft labels. As the backbone network, we used ResNet101 [8] and incorporated IN layers at the ends of three residual blocks, following IBNNet_b [19]. We applied the same hyperparameters to [2] except for the number of iterations. All training (including baseline models) was done with four Tesla V100 GPUs. The parameters of the backbone network were initialized with the ResNet101 pre-trained on ImageNet except for the IN layers. The IN layers, RPN, and RoIhead were initialized with random parameters. During the inference (testing) phase, we used the teacher network.

On the car-mounted camera dataset, the detectors were trained with 20,000 and 40,000 iterations for the pretraining and the student-teacher learning of SS-DGOD (or WS-DGOD), respectively. During the training, we saved the models and evaluated the performance on the validation at every 4,000 iterations, and the best model was used for the evaluation. The whole training took about two days. For fair comparisons, the compared models on Single-DGOD and DGOD were trained with 60,000 iterations, and the best models at the validation of every 4,000 iterations were used for evaluation.

On the artistic style image dataset, we performed the same procedure for training, validation, and evaluation, but the numbers of iterations for the pretraining and the student-teacher learning were set to 10,000 and 20,000 respectively, and the validation was conducted at every 2,000 iterations. The whole training took about one day. For fair comparisons, the compared models on Single-DGOD and DGOD were trained with 30,000 iterations, and the best models at the validation of every 2,000 iterations were used for evaluation.

5.3 Results

Table 2 shows the results on the car-mounted camera dataset. We evaluated with the mean Average Precision (mAP50) when the IoU threshold was 0.5. For fair comparisons, the backbone networks were ResNet101 for all of the methods. The mAP50 was increased to (30.3, 21.3) by the student-teacher learning on the unlabeled domains (SS-DGOD) although that of the baseline model (Gaussian FasterRCNN) trained only on the labeled domain (daytime-sunny) was (25.3, 13.3). IN layers improved the performance to (37.6, 24.0). In addition, the refinement process in Eq. 3 by the weak labels on WS-DGOD further boosted the mAP to (38.7, 25.3). This result significantly exceeds (32.3, 18.7), which is the result of the state-of-the-art method on Single-DGOD [25]. Also, this result is better than those of the models trained with supervised learning on the three domains (DGOD). Incorporating IN layers on supervised learning (i.e., equivalent to IBNNet_b [19]) does not always improve the performance on night-rainy. In contrast, IN layers constantly improve the performance on SS-DGOD and WS-DGOD by suppressing the bias on the labeled domain in student-teacher learning.

Figs. 3 and 4 show the qualitative comparisons on dusk-rainy and night-rainy, respectively. We observe that false negative detection of the baseline model was drastically improved by SS-DGOD and IN layers.

Table 3 shows the results on the artistic image style dataset. Similar to Table 2, SS-DGOD and WS-DGOD settings significantly improved the performance, compared with Single-DGOD setting, and IN layers constantly boosted the performance on SS-DGOD and WS-DGOD settings. Those results are comparable to those of the models trained on DGOD and Oracle settings. Also, the models trained on the proposed SS-DGOD and WS-DGOD settings performed comparably to or better than those on UDA-OD setting although ours do not use the target domain data during the training.

Fig. 5 shows the qualitative comparison on watercolor. We can see in these examples that false negative detection and wrong regression were improved by SS-DGOD and IN layers.

6 Conclusions

In this paper, we proposed two new problem settings called semi-supervised domain generalizable object detection (SS-DGOD) and weakly-supervised DGOD (WS-DGOD) to train object detectors that can generalize to unseen domains. Compared with the existing DGOD settings, their advantage is that the labeled data from multiple domains are not required for training because the detectors are trained with the labeled data from one domain and unlabeled (or weakly-labeled) data from multiple domains. In contrast to unsupervised domain adaptive object detection (UDA-OD) settings, they do not need the target domain data during the training. Therefore, they are more practical than the existing problem settings.

We showed that the object detectors can be effectively trained on the proposed two settings with the same student-teacher learning framework. Also, we proposed incorporating instance normalization (IN) layers into the detection networks to suppress the bias to the labeled domain data. The experimental results showed that the detectors trained on SS-DGOD and WS-DGOD performed significantly better than the baseline models. It was also confirmed that IN layers improved the performance of the detectors on the two proposed settings. In addition, they performed comparably to or better than the models trained on UDA-OD setting while they did not use the target domain data during the training.

In future work, we are planning to investigate the performance of recent Transformer-based detectors on SS-DGOD and WD-DGOD settings.
